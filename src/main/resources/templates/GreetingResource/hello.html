Simple String

Code: \{i18n:hello\})

Result: {i18n:hello}

Simple String With Dot

In Quarkus when using a dot in a property you have to put it in ['']

Code: \{i18n:['hello.simple']\})

Result: {i18n:['hello.simple']}

String With Java Format

You can combine multiple namespaces to use the str:format in combination with the i18n namespace

Code: \{str:format(i18n:hello-str, name)\})

Result: {str:format(i18n:hello-str, name)}

String With Message Format

You also create a namespace for the Java message format (not very performant because no caching is done).

Code: \{mf:format(i18n:['hello.with-format'], name)\})

Result: {mf:format(i18n:['hello.with-format'], name)}

String With ICU Message Format

ICU message format is the standard way for translating complex messages (also not performant).

icu4j ist not compatible with native

Code: \{icumf:format(i18n:['hello.with-icu-format'], name, age)\})

Result: {icumf:format(i18n:['hello.with-icu-format'], name, age)}

Result: {icumf:format(i18n:['hello.with-icu-format'], 'Qute', 1)}

Type-safe Message Bundles

Quarkus gives you a type-safe way to access messages which is really nice. But there are some problems:

  • there is no possibility for message format (you can use Qute template syntax as an alternative)
  • because the default message is in the Java code translation tools cannot use them
  • you cannot have dots in the message key

Code: \{msg:hello_name(name)\})

Result: {msg:hello_name(name)}

Code: \{msg:hello_name_age(name, age)\}

Result: {msg:hello_name_age(name, age)}
